# Trilha Backend por Nivel

## Objetivo
Organizar o estudo em fases claras para evoluir de junior para senior com foco em backend de mercado.

## Nivel Junior (Base Solida)
### Objetivos
- Entender como uma API funciona ponta a ponta.
- Conseguir implementar CRUD com qualidade basica.

### Competencias tecnicas
- HTTP, JSON, REST e status codes
- POO e organizacao em camadas (Controller, Service, Repository)
- SQL basico/intermediario (joins, filtros, ordenacao)
- Validacao de entrada e tratamento de erros
- Git basico e fluxo de branch
- Testes unitarios basicos

### Entregas praticas esperadas
- Criar endpoints com validacao e respostas coerentes
- Persistir e consultar dados com SQL limpo
- Cobrir regras principais com testes unitarios

### Sinais de prontidao para Pleno
- Consegue explicar o por que das decisoes de codigo
- Nao depende de copia e cola para tarefas comuns
- Consegue corrigir bugs de baixa/media complexidade

---

## Nivel Pleno (Consistencia e Escala)
### Objetivos
- Projetar solucoes com melhor manutenibilidade.
- Aumentar confiabilidade e performance.

### Competencias tecnicas
- SOLID aplicado ao codigo real
- Design patterns (Factory, Strategy, Adapter, Repository)
- SQL avancado e tuning com EXPLAIN
- Transacoes, locks e concorrencia no banco
- Testes de integracao e testes de API
- Seguranca aplicada (JWT, autorizacao por permissao, OWASP)
- Cache, paginacao e idempotencia
- CI/CD basico e ambiente com Docker

### Entregas praticas esperadas
- Refatorar modulo legado sem regressao funcional
- Melhorar latencia de endpoints com evidencia de medicao
- Endurecer endpoints criticos contra erros e abuso

### Sinais de prontidao para Senior
- Consegue quebrar problema grande em entregas pequenas
- Propõe trade-offs tecnicos com clareza
- Ajuda o time a definir padroes e boas praticas

---

## Nivel Senior (Arquitetura e Lideranca Tecnica)
### Objetivos
- Tomar decisoes arquiteturais sustentaveis.
- Garantir evolucao do sistema com foco em negocio.

### Competencias tecnicas
- System design (escalabilidade, consistencia, disponibilidade)
- Arquitetura orientada a eventos e mensageria
- DDD aplicado por contexto
- Observabilidade completa (logs, metricas, tracing)
- Estrategia de versionamento de API
- Gestao de risco tecnico e plano de rollback
- Governanca de qualidade (testes, review, padroes)

### Entregas praticas esperadas
- Desenhar e implementar evolucao arquitetural incremental
- Definir estrategia de observabilidade e incidentes
- Guiar decisoes de performance, custo e prazo

### Sinais de senioridade consolidada
- Resolve causa raiz, nao apenas sintoma
- Equilibra qualidade tecnica com impacto de negocio
- Multiplica conhecimento e melhora autonomia do time

---

## Plano de execucao sugerido (12 semanas)
### Fase 1 (Semanas 1-4) - Junior forte
- Fundamentos, REST, SQL, testes unitarios

### Fase 2 (Semanas 5-8) - Pleno pratico
- SOLID, patterns, SQL avancado, integracao/API tests, seguranca

### Fase 3 (Semanas 9-12) - Senior em construcao
- System design, observabilidade, mensageria, DDD

## Regra de evolucao
So avance de nivel quando conseguir entregar os itens praticos com evidencia:
- codigo
- testes
- medicao
- documentacao curta de decisao tecnica
