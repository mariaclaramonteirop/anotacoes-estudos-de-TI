# Checklist Semanal - Evolucao Backend

## Como usar
- Marcar no final de cada semana
- Se 2 ou mais itens ficarem nao concluidos, repetir foco na semana seguinte

## Semana 1 - Banco e SQL
- [ ] Mapeei queries mais usadas no modulo estudado
- [ ] Rodei EXPLAIN nas queries criticas
- [ ] Criei ou revisei indices com justificativa
- [ ] Comparei antes/depois com evidencias
- [ ] Documentei trade-offs

## Semana 2 - Testes
- [ ] Tenho testes unitarios das regras principais
- [ ] Tenho testes de integracao com banco
- [ ] Tenho testes de API para sucesso
- [ ] Tenho testes de API para erro
- [ ] A suite ajuda a refatorar com seguranca

## Semana 3 - Seguranca
- [ ] Revisei autenticacao e expiracao de token
- [ ] Revisei autorizacao por permissao/escopo
- [ ] Endureci validacao de entrada
- [ ] Cobri riscos do OWASP aplicaveis
- [ ] Configurei controle basico de abuso (rate limit)

## Semana 4 - Observabilidade
- [ ] Logs estruturados estao padronizados
- [ ] Existe correlacao de requisicoes
- [ ] Metricas de latencia e erro estao disponiveis
- [ ] Alertas minimos foram definidos
- [ ] Existe guia de resposta a incidente

## Consolidacao final (Dias 29-30)
- [ ] Entreguei 1 modulo com melhoria de SQL
- [ ] Entreguei cobertura de testes do fluxo escolhido
- [ ] Entreguei reforco de seguranca
- [ ] Entreguei instrumentacao de logs e metricas
- [ ] Entreguei documentacao tecnica clara

## Autoavaliacao final
- [ ] Consigo explicar o por que das decisoes
- [ ] Consigo medir o impacto das mudancas
- [ ] Consigo reproduzir e corrigir falhas com metodo
- [ ] Consigo defender trade-offs tecnicos e de negocio
